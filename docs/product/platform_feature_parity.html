Implementation Snapshot Android iOS Web

Platform Feature Parity

Hopply capability coverage across Android, iOS, and the Kotlin/JS web host. This matrix reflects wired functionality in the repository as of May 25, 2026, including features available behind Feature Lab flags or dogfood tooling.

✓ Supported in source — Not supported By design Intentional platform scope Flagged / variant Availability is gated
Web guest users are intentionally not supported. Web requires a permanent account and does not expose anonymous sign-in or guest-to-account upgrade. This is a product-scope decision, not an outstanding parity defect.

Open Parity Work

The matrix identifies these remaining functional gaps:

  • Web attachments: implement browser file/blob persistence, hashing, open/download behavior, and transfer scheduling before enabling attachment controls.
  • Web export: add JS targets and browser download/delivery UX for export generation and history.
  • iOS guest upgrade: wire the permanent-account linking provider for anonymous mobile users.
  • Web integrity recovery UI: expose the local database recovery path currently present in the mobile app shell.

Guest support on web is deliberately excluded and is therefore not listed as parity work.
